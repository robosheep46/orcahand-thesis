# 🐞 调试案例库

> 动手过程中遇到的**真实问题**——装机、接线、标定、环境、训练、复现——按「现象 → 定位 → 根因 → 修复 → 复盘」沉淀。**调试时由 AI 协助直接落盘**，事后翻库不重复踩坑。

## 这个库装什么

| 收 | 不收 |
|---|---|
| 硬件：舵机不响应、标定行程错、接线/串口、绞盘打滑、打印件干涉 | 纯选型决策（→ [[面试知识库/项目决策/index\|项目决策]]） |
| 仿真：WSL2/MuJoCo 安装、MJCF 加载报错、训练不收敛、reward 调参 | 体系化原理（→ [[面试知识库/index\|面试知识库]]） |
| sim2real：真机零样本翻车、gap 现象 | 单纯的"今天做了啥"流水账（→ [[执行计划]]） |

## AI 写入约定

> 给在本库开 Claude Code 的会话：当一次调试**定位到根因并验证修复**后，主动把它落成一篇案例。

1. 复制 [[调试案例库/_模板\|_模板.md]] → 新文件 `调试案例库/<短标题>.md`
2. 打 tag：`硬件` 或 `仿真`（或 `sim2real`）+ 涉及的器件/工具（如 `STS3215`、`MuJoCo`）
3. 在下面的「案例索引」加一行
4. 触发时机：**根因清楚 + 修复已验证** 才写；编译错误、接线松动、纯手滑不写
