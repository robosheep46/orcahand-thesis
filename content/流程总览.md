# 🔧 流程总览（操作层）

> [[执行计划]] 回答**何时做什么**（P0–P5 里程碑、采购、风险）；本篇回答**每条流水线怎么跑**——输入 / 管线 / 命令 / 现状 / 验收。两篇互链：战略 ↔ 操作。
>
> 现状图例：✅ 已就绪　🔨 在改/半成品　⬜ 待办

## 全局图

```
软件线（零硬件依赖，现在就能推）
  遥操(A) ──采集(B)──> LeRobot 数据集 ──训练──> IL 策略(C) ┐
                                                          ├─> 仿真验证 ─┐
                       sim_rl ───训练@4090──> RL 策略(D) ┘               │
                                                                        交汇：零样本上真机
硬件线（被舵机到货卡住）
  舵机到货 → 单指 → 三指 → 标定(E) ───────────────────────────────────┘ → 真机遥操/sim2real(F)
```

**关键决策**：遥操**不自造**，接官方 `orca_teleop` 全家桶（MediaPipe→重定向→仿真/真机共用同一套 + gRPC + 录数据集）。本仓 `teleop/teleop_sim.py`（手调 curl 启发式）是探路 PoC，**已被官方栈取代**，仅留作理解参考。

---

## 流程 A — 遥操（接官方 orca_teleop）

| 项 | 内容 |
|---|---|
| 输入 | USB 摄像头 → MediaPipe 手部 21 关键点 |
| 管线 | `orca_teleop`：MediaPipe publisher → gRPC ingress → `orca_retargeter`（优化式重定向，pytorch_kinematics 梯度匹配人手指尖↔机械手指尖） → Sink（`SimSink` 驱动 orca_sim ／ 真机 `OrcaHand`） |
| 跑通命令 | `python scripts/teleop_sim.py --env right --local --show-video` |
| 现状 | ✅ 官方代码 `repos/orca_teleop`、`repos/orca_retargeter`；🔨 本仓 PoC（弃） |
| 三指适配（真要做的活） | 重定向器吃 `model_path`（urdf + config.yaml + hand_scheme.yaml）。**三指 URDF 缺**（只有 v2 五指 URDF）。两条路：① 用五指 URDF 跑，`loss_coeffs` 把无名/小指设 0，只取拇/食/中关节驱动我们 10 电机（**最快，先走这条**）；② 造三指 URDF |
| 验收 | 摄像头开合 → 仿真三指实时跟随，延迟 <100ms（执行计划 P2） |

## 流程 B — 数据采集

| 项 | 内容 |
|---|---|
| 管线 | `orca_teleop/scripts/record_dataset.py` 录 (obs, action) → 对齐 LeRobot 格式；`replay_dataset.py` 回放校验 |
| 任务 | 带物体场景里录抓握/搬运示教（立手 grasp 场景或加物体） |
| 现状 | ✅ 官方 `record_dataset.py`/`replay_dataset.py`；⬜ 三指任务场景待定 |
| 验收 | 录够 N 条、回放无误、格式对齐 LeRobot |

## 流程 C — 模仿学习（保底线 · P3）

| 项 | 内容 |
|---|---|
| 管线 | LeRobot 数据集 → **ACT / Diffusion Policy** 训练 → eval |
| 现状 | 🔨 现有 `il/` 是玩具 BC，且任务退化（见案例）；⬜ ACT/DP 待上 |

## 流程 E — 硬件装机（P0→P1）

```
舵机到货 → configure_motor_chain.py 配 ID/波特率
        → check_motor.py 单个验证
        → 单指(3 舵机)装 + orca_core 跑通官方脚本
        → 三指装
        → calibrate.py 逐关节找行程标定
```

- 现状：✅ orca_core 控制栈/标定脚本齐；✅ STS3215×4 + FE-URT-2 + 轴承/腱绳部分已到；⬜ 打印件待舵机到货卡尺核孔距后代打

## 流程 F — 真机遥操 / sim2real（交汇）

- 真机遥操 = 流程 A **换个 Sink**（`SimSink` → `OrcaHand`），官方同一套，几乎零额外开发
- sim2real 两次触点（见 [[执行计划]] §3.1）：M3末 粗暴零样本试水 → P4 域随机化系统攻坚 + gap 定量分析

---

## 关键路径 & 优先级

1. **现在（无硬件）**：流程 A 跑通官方遥操（五指先验证全链路）→ A 三指适配 → 流程 B 录数据 → 流程 C 上 ACT/DP。**这条链把 P2 + P3 保底线全干了，完全不等硬件。**
2. **舵机到货**：并行起流程 E 装机。
3. **4090 到位**：流程 D 主线 RL。
4. **两线就位**：流程 F 交汇，零样本上真机。
