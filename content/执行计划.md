## 0. 课题定案

**题目（暂定）**：《低成本三指灵巧手平台的构建与基于学习的操作技能研究》

**一句话**：把 ETH ORCA 开源手（飞特 STS3215）裁剪成三指，做一只**重实用、重力控**的手——不拼炫技（翻方块/转笔），而是把收纳/抓取/放置/握手这类日常操作做好；硬件线与仿真线双轨，跑通 遥操作 → 模仿学习（保底）→ RL sim2real（主线），并沿**力控三档阶梯**逐级逼近真力控。

**定位**：嵌入式岗 + 算法加分，受众机器人公司。

**🔀 定位转向（2026-06-10）：从「灵巧」到「顺应」**。市面灵巧手卷运动学炫技（in-hand 再定向），但日常高频的是收纳/抓取/放置/人机交互，共同内核是**力控/顺应**（抓鸡蛋不碎、轻放不砸、握手可被推退让）。贡献点从"运动学灵巧度"换成"力域能力"——对低成本平台更稀缺、更有新意、更可量化评测。详见 [[面试知识库/项目决策/index|项目决策]]。

**论文贡献点（v3）**：
1. **低成本三指平台的力控/顺应能力**——力控三档阶梯，串行带 gate：Tier 0 软件限力（STS3215 的 `Torque Limit` + 电流反馈）→ Tier 1 低成本指尖触觉 → Tier 2 单指 FOC 真力矩（**核心贡献**）
2. 力矩 vs 位置动作空间的 sim2real 对照实验（仿真内完成，ORCA 论文未做）
3. 低成本（≈¥2400）三指平台上 IL + RL sim2real 全链路复现与 gap 分析（reward/loss 内含力维度）
4. 单指 FOC 可反驱执行器对比研究（= Tier 2，由原"可选冲高项"升为力控阶梯顶档）

## 1.5 工作架构（双机双仓 + 端到端流程，2026-06-10）

**双机职能**（弱机核显跑不动渲染/训练 → 职能由硬件决定）：

| | 🧠 强机（训练/仿真机） | 🤖 弱机（控制/部署机） |
|---|---|---|
| 仓库 | `orcahand-sim`（新建） | `dexterous-hand`（本仓瘦身后） |
| 干什么 | MuJoCo 渲染、RL/IL 训练、数据集、出 `policy.ckpt` | 实时控制环、手势遥操、装机标定、实物采集、策略部署推理 |
| 重依赖 | torch / mujoco / gymnasium | pyserial / numpy / mediapipe |

> 数据流成环：强机训出 `policy.ckpt` → 弱机部署；弱机遥操/采集 `dataset` → 强机训练。
> 接口不变靠 `orca_core` 的 `MotorClient` 抽象层——FOC（Tier 2）以后实现同一接口即可挂入，上层 RL/IL/遥操不改。

**端到端任务流程（1→6，标机器）**：
1. **Tier 0 力控底座验证**（单舵机读电流 + 限力软抓取）— 控制机 ← 当前
2. 装三指手 + 标定 + 控制环跑通 — 控制机
3. 手势遥操采集（MediaPipe→retarget→真手，**录力轨迹**）— 控制机
4. 仿真 + 训练（MJCF 三指，RL 力控 reward / IL 含力 loss）— 训练机
5. sim2real 部署 + **力控评测**（策略下发控制机，跑抓鸡蛋评测表）— 双机
6. 冲高（gate 解锁）：Tier 1 触觉 / Tier 2 FOC 单指 — 双机

### 3.1 仿真线 → sim2real 细化 roadmap

> **sim2real 不是一个时间点，是两次触点**：先试水（M3末）、后攻坚（P4）。周期锚定 2026-07~2027-06。

| 阶段 | 月份 | 仿真线做什么 | sim2real |
|---|---|---|---|
| 现在 | 2026-06（提前） | ✅ 环境 + 三指 MJCF + 动力学验证（抓握、10 关节跟踪） | — |
| P0 | M1 / 07 | 接 RL 管线，**复现五指转方块基线**（证明能训起来） | — |
| P1 | M2 / 08 | **三指包成 gym 任务 + 三指重训** → 答「三指能否手内操作」⭐贡献点1 | — |
| P1末 | M3 / 09 | **力矩 vs 位置动作空间对照实验**（仿真内）⭐贡献点2 | — |
| 会师 | M3末 / 09–10 | — | 🔵 **首次试水**：三指真机装好+仿真有策略 → **零样本上真机看 gap** |
| P3 | M3.5–5.5 / 10–12 | 模仿学习保底线（ACT/DP） | — |
| P4 | M5–8 / 12–02 | — | 🔴 **正式攻坚**：域随机化系统训练 + 真机零样本/微调 + **gap 定量分析** ⭐贡献点3 |

**为什么 sim2real 排在 M5–8**：要三者同时就位——① 真机三指装好（硬件线 M2 后）；② 仿真有能用的 RL 策略（M2–3 训出）；③ 域随机化做足（跨 gap 的关键）。中间用 **P3 模仿学习当保底**：主线翻车也能让毕设成立。

**两次触点区别**：M3末**首测**=粗暴零样本，目的是早知道 gap 多大、坑在哪（批量买齐整手前的关键信息）；P4**正式**=系统化域随机化+微调+定量 gap，论文要写的结果。
