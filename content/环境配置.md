# 🛠 环境配置

> 这个项目**几乎全程 Windows 原生**就能干——MuJoCo、CadQuery、舵机控制、遥操作都不用虚拟机。唯一要 Linux 的是 Isaac RL 训练，跑在 **AutoDL 云**（远程 Linux），不是本地虚拟机。本地无需安装 WSL2。

## 谁跑在哪（一图流）

| 环节 | 跑哪 | 要 Linux/VM 吗 |
|---|---|---|
| MuJoCo / MJCF（裁三指、看模型） | Windows 原生 | ❌ 不要 |
| CadQuery 出件（基座/绞盘） | Windows venv | ❌ 不要 |
| orca_core 控舵机（FeetechClient + 串口） | Windows 原生 | ❌ 不要（COM 口直插，WSL2 反而要折腾 USB 透传） |
| 遥操作 MediaPipe（摄像头→retargeter） | Windows 原生 | ❌ 不要 |
| PyTorch / ACT / DP 写码 + 小数据调试 | Windows 原生 | ❌ 不要 |
| **orca_sim 仿真环境（MuJoCo+Gymnasium）** | Windows 原生 | ❌ 不要 |
| RL 训练**提速**（嫌本地 CPU 慢时） | AutoDL 云 4090 | ✅ 用云不用本地 VM |
| 大规模 IL 训练 | AutoDL 云 | ✅ 同上 |

> 仿真环境本身 MuJoCo 跑 CPU、本地就能跑；上云只为**训练提速**，不是必需。本地 WSL2 全程不用。

## 已验证的安装步骤

### MuJoCo（Windows 原生）✅ 已跑通

```powershell
# 用系统 Python 3.14 直接装
& "C:\Users\12415\AppData\Local\Programs\Python\Python314\python.exe" -m pip install mujoco
# 看模型（拖 scene 文件进 viewer）
& "...Python314\python.exe" -m mujoco.viewer --mjcf=scene_3f_right.xml
# 无头校验装配数（CI/脚本用）
python -c "import mujoco; m=mujoco.MjModel.from_xml_path('scene_3f_right.xml'); print(m.nu, m.njnt)"
```

- 实测版本：mujoco **3.9.0**，Python 3.14，免显卡（离屏渲染默认帧缓冲 640×480，要更大在 XML 加 `<visual><global offwidth=.../></visual>`）。
- 渲染存图还需 `pip install pillow`。
- ⚠️ **MuJoCo 不支持非 ASCII（中文）路径**：工程在中文目录下会报 `ParseXML: Error opening file`。毕设源文件夹已用纯英文 `dexterous-hand`；**要喂给 MuJoCo 的工程别放回中文目录**（见 [[调试案例库/MuJoCo中文路径打不开模型]]）。

### orca_sim 官方仿真环境（Windows，纯英文路径）✅ 已跑通

官方仿真环境，**MuJoCo + Gymnasium，不是 Isaac**——本地 Windows 原生跑，免云免显卡。装在项目内 `dexterous-hand\repos\orca_sim`，venv `dexterous-hand\sim_env\.venv`（Python 3.11）：

```powershell
cd C:\Users\12415\Desktop\dexterous-hand
uv venv --python 3.11 sim_env\.venv
uv pip install --python sim_env\.venv\Scripts\python.exe -e repos\orca_sim pillow
# 冒烟：随机动作跑右手
cd repos\orca_sim
...\sim_env\.venv\Scripts\python.exe random_policy.py --env right --render-mode rgb_array --steps 3
```

- 环境变体：`right/left/combined` + `_extended`（含相机/U2D2/风扇）+ **`right_cube_orientation`**（手内转方块 = RL 基线任务，obs 含方块姿态）。
- 五指右手 obs (34,) / action (17,)；遵循 Gymnasium API（`reset`/`step`/`action_space`），接任何 gym 兼容 RL 库（stable-baselines3 / cleanrl）。
- 实测版本：orca-sim **0.1.0**、gymnasium 1.3、mujoco 3.9。

### CadQuery（Windows，独立 venv）✅ 已跑通

CadQuery 依赖 OCP（OpenCascade 绑定，几百 MB），需 Python 3.11/3.12（3.14 无 wheel）。用 uv 的 3.11 建独立 venv（实测 cadquery **2.7.0**）：

```powershell
cd C:\Users\12415\Desktop\dexterous-hand
uv venv --python 3.11 cad_3f\.venv
uv pip install --python cad_3f\.venv\Scripts\python.exe cadquery
# 出件
cad_3f\.venv\Scripts\python.exe cad_3f\feetech_plate.py   # → FeetechPlate.stl + coupon.stl
```

- CAD 工作一律用 `cad_3f\.venv`（系统 Python 3.14 装不了 CadQuery）。
- 当前产物（`dexterous-hand/cad_3f/`，参数在各脚本顶部）：
  - `feetech_plate.py` → `FeetechPlate.stl`（STS3215 三工位基座）+ `coupon.stl`（单工位测试块）。舵机贴板背面、输出轴+绞盘从让位孔伸到正面。安装孔取自 STEP（机身中心 ±19.5×±9.5），输出轴偏置 10.1。
  - `capstan_spool.py` → `CapstanSpool.stl`（腱驱绞盘 ⌀21×6，绕线芯 ⌀15；4×M3 沉孔对主舵盘 ⌀14 节圆，套金属舵盘，不印塑料花键）
- 两处到货收口（先打 `coupon.stl` 试插确认再打整板）：基座角孔距（STEP 噪声大 ±1mm，卡尺量准）；让位孔是否过整只绞盘（⌀22）还是只过腱绳（⌀8）。

### STS3215 官方 CAD 与关键尺寸（参考）

官方 CAD 包：`D:\BaiduNetdiskDownload\ST-3215-C001\`（含 2D 尺寸图、主/副舵盘图、25T 花键图、3D STEP `ST3215-...stp`、官方 U 支架图）。

| 接口 | 值 | 出处 |
|---|---|---|
| 机身 L×W×H | 45.23 × 24.73 × 35（STEP 确认） | st3215-2D尺寸图 / STEP |
| 输出花键 | 25T，⌀5.9 铜齿，⌀6 外缘 | 5.9MM-25T尺寸图 |
| **金属主舵盘** FK-AP-019-25T | 外径 ⌀19.95，**4×M3 在 ⌀14 节圆**，中心凸台 ⌀9，厚 4.5 | 主舵盘图 |
| 端子 | 5264 / 2.54mm 3P（SIG/V/G） | 2D尺寸图 |

> 绞盘按"套金属主舵盘 + 4×M3 拧紧"设计；主舵盘 4×M3 在 ⌀14 与官方绕线芯 ⌀14 重合，故绞盘绕线芯放大到 ⌀15 让开螺孔。

### orca_core 控舵机（Windows 原生）📋 待舵机到货验证

- Windows 原生，靠 pyserial 走 COM 口，比 WSL2 省事（不用 USB 透传）。

- 单指验证：3× STS3215（7.4V）+ FE-URT-1 + 7.4V 电源 → 跑官方脚本。详见 [[执行计划]] P0。
