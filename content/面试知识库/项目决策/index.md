# 🧩 项目决策

> 每个关键选择**为什么这么定**——答辩官最爱追问的就是这些。每条决策写清：选项 / 取舍依据 / 代价 / 如果重来。完整决策记录见 [[执行计划]]。

## 定位转向：从「灵巧」到「顺应」（2026-06-10 定）

> 答辩开场要回答的第一问：**你这手到底解决什么问题？** 不跟炫技路线卷。

- **重新定义**：市面灵巧手多在比 in-hand manipulation（翻方块 180°、转笔）——本质是**运动学再定向**，好出 demo、RL 好刷，但生活实用价值低。日常真正高频的是**收纳 / 抓取 / 放置 / 人机交互（握手）**，共同内核是同一件事——**力控 / 顺应性（compliance / force regulation）**：抓鸡蛋不碎、放置不砸、握手安全可被推动。
- **贡献点从「运动学灵巧度」换成「力域能力」**。后者对**低成本平台**反而更稀缺、更有新意——市面能做好力控的手基本都贵（带指尖触觉 / 力矩传感）。**「用便宜舵机做出有用的顺应抓取」本身就是干净的 contribution**。
- **代价 / 边界**：放弃"高灵巧度"叙事；"实用性"必须**操作化成可评测指标**（不损坏率、接触力峰值、力受限、可反驱性），否则答辩被打。
- **管线不浪费**：遥操 / IL / RL sim2real 全保留，只往 reward / loss / sim 里**加力维度**（reward hacking 经验复用）。

## 力控能力阶梯（带 gate 的三档，串行推进）

> "三档都要"，但**必须串行**：每档拿出数据才解锁下一档；FOC 永远只做 1 指 PoC，绝不全手——否则毕设沦为执行器研发。

| 阶段 | 做什么 | 解锁 gate（数据说话） | 论文里是什么 |
|---|---|---|---|
| **A · Tier 0** 软件力控 | Torque Limit(addr 48) 实时限力 + 电流读数搭外部粗顺应；软抓取 / 限力 / 握手退让 | 抓鸡蛋模型不碎率、握手接触力封顶 | 主贡献①：低成本舵机的软件级顺应 |
| **B · Tier 1** 指尖触觉 | 加 FSR / 气压 MEMS 指尖力敏，补亚牛顿轻放 | 轻放接触冲击力峰值↓、细力任务成功率↑ | 贡献②：低成本触觉 + 力控融合 |
| **C · Tier 2** FOC 原型 | **只做 1 根手指** BLDC + 磁编 + FOC，验真力矩 / 可反驱 | 单指力带宽、可反驱性 vs STS3215 对比曲线 | 贡献③ / 未来工作：可反驱执行器 PoC |

**三条铁律**：① 串行不并行（A 无数据不碰 B）；② Tier 2 永远 1 指 PoC；③ 已有管线全保留只加力维度。

### STS3215 力域能力（Tier 0 的硬件底牌，实测寄存器）

- **限力矩** `Torque Limit` addr 48（RAM，可实时写，0–1000=0–100%）→ **软抓取金矿**：压低力矩上限，抓到物体自然停在该力，不需力闭环就能抓鸡蛋不碎。
- **读电流** `Present Current` addr 69（1 LSB = 6.5 mA）→ 能读但**粗**，受齿轮摩擦主导，用于粗打滑 / 接触检测。
- **读负载** `Present Load` addr 60（10 位幅值+方向，是 **PWM 占空比**，非真实力矩）。
- 过流保护 addr 28；模式 addr 33（0 位置 / 1 恒速 / 2 PWM 开环 / 3 步进）。
- **关键边界**：**无原生电流/力矩闭环模式**，力闭环须外部自搭（读电流→估力→调位置/限力），带宽受串行总线限（共享菊花链，几百 Hz 封顶）。亚牛顿轻放靠 STS3215 难——这正是 Tier 1 触觉要补的。

## 已定案的关键决策

### 执行器：STS3215 ×11

- **为什么**：¥100/个、LeRobot 同款生态成熟、`orca_core` 原生兼容、自带磁编 + 负载反馈，单舵机集成位置环
- **代价**：力矩 1.9 N·m 偏小、外形与官方塔不配 → 需重设计基座
- **待写**：和 Dynamixel 的对比；为什么不用无刷 + 外置驱动

### 五指 → 三指裁剪

- **为什么**：成本 / 装配 / 训练复杂度全降；三指足以覆盖捏取与基础手内操作
- **风险**：三指能否完成手内操作（in-hand manipulation）——**仿真先行验证**后再批量采购
- **待写**：三指可行性的文献/仿真依据；和"对置拇指最小手"的关系

### 三指 → 五指 / 触觉：向上兼容（架构是五指的子集）

> 答辩高频问"为什么只做三指、能不能扩"。结论：**三指是五指的子集，向上兼容，扩展是增量、不推翻**。

- **仿真**：三指由官方五指模型删 ring+pinky 得到，**原始五指 MJCF 完整保留**；orca_sim 的 `OrcaHandRight` 本就是五指环境。回五指=用回原文件；触觉=MuJoCo 加 touch sensor。
- **机械**：手掌 `R-Carpals` 是完整官方件，五指安装位都在，三指只是少装两根。手指/手掌/模具 STL 全官方原件。
- **要重做但便宜**：电机基座 `FeetechPlate`（参数化，`N_SERVOS` 3→16 重打）、绞盘多打几个。
- **电气增量**：10→16 电机（补买 6 个 STS3215）；FE-URT-2 一根总线挂上百舵机不用换，但要换更大电源。
- **触觉**（P5 冲高项）：仿真侧加 touch sensor 简单；硬件侧唯一要改的是**指尖件留传感器凹槽**。
- **保持兼容只需守两条**：① 始终用完整官方手掌；② 基座/绞盘保持参数化。
- **诚实边界**：五指真机难度不在结构（兼容），在**腱绳布线更挤、16 关节标定更繁、RL 维度更高**——是"更费时"不是"重来"。

### 传动：平板基座 + 绞盘腱鞘

- **为什么**：STS3215 外形塞不进官方塔；绞盘 + 腱让电机位置自由布局
- **代价**：引入腱的弹性/迟滞，标定更复杂
- **待写**：腱传动的回差/张力管理
